# 층간소음 측정 앱 설계 (Android Kotlin + NDK)

- 작성일: 2026-06-20
- 범위: **측정 앱 (NDK 코어 + 출시용 UI/수익화)** — 전체 서비스 4개 시스템 중 1번째
- 스택: Android Kotlin + NDK(C++), Jetpack Compose, Room, Google Maps SDK, Google Play Billing, AdMob

---

## 1. 배경 및 목표

기존 Flutter 프로토타입은 휴대폰 자체 오디오 프로세싱(AGC 등)과 Flutter의
GC·부동소수점 딜레이로 측정 신뢰도 이슈가 있었다. 이를 해결하기 위해 **순수
Kotlin 네이티브 + NDK** 로 재구성한다. UI는 Kotlin/Compose, 오디오 수집과
DSP 연산은 전부 C++ 네이티브에서 처리해 OS 보정이 없는 Raw Audio를 안정적으로
확보하는 것이 핵심이다.

측정 단위는 **국제 표준(IEC 61672)** 인 A-가중 dB(A) 기반으로 하여, 한국뿐
아니라 해외(미국·일본·스페인·동남아 등)로도 동일 로직·다국어로 확장 가능하게
한다.

## 2. 빌드 범위 및 순서

이 문서는 **출시 가능한 측정 앱 전체**를 정의하되, 구현은 아래 순서로 한다.

1. **측정 코어(NDK)** — 오디오 수집·dB(A)·Leq/Lmax·링버퍼·WAV 저장 *(최우선)*
2. **앱 화면(Compose)** — 측정 흐름 전 화면 + 로컬 저장(Room) + 지도/GPS
3. **수익화** — Play Billing(무료/프로) + AdMob(무료 광고)
4. *(이후 단계, 이 문서 범위 밖)* Firebase 연동 → 모니터링 웹 → 더미 AI

### 이 문서 범위 밖 (다음 프로젝트)
- Firebase 업로드 / SHA-256 무결성 해시 / 시리얼별 캘리브레이션 파일 동기화
- 로컬 웹서버 스트리밍, 모니터링 웹, 실제 AI 모델
- 원격 제어, 레포트 생성·신청 폼

## 3. 화면 흐름 (UX)

```
0 언어 선택 → 1 측정 전(준비) → 2 측정 중 → 3 정지 후 저장
        → 4 이벤트 목록 → 4-1 이벤트 상세 → 5 결제(무료/프로)
```

| # | 화면 | 핵심 요소 |
|---|---|---|
| 0 | 언어 선택 | 진입 시 언어 선택(검색 + 스크롤 목록). 한·영·일·중·스페인·베트남·태국·인니·필리핀·말레이 등 |
| 1 | 측정 전(준비) | 상태 체크(마이크 연결·캘리브레이션·GPS 위치·기기 수평), 임계값, "측정 시작" |
| 2 | 측정 중 | 실시간 dB(A) 게이지, Leq/Lmax/임계, 경과시간, 이동감지, "측정 정지" |
| 3 | 정지 후 저장 | 세션 요약(시간·이벤트 수·Lmax·Leq), 측정 위치(지도), 메모, 원음 보관 체크, 저장/삭제 |
| 4 | 이벤트 목록 | 저장된 이벤트(시각·dB·태그·노트·이동플래그), 재생 횟수 표시, (무료)광고 배너 |
| 4-1 | 이벤트 상세 | 파형+재생 플레이어, 위치(지도), 태그(+AI 추천 자리), 노트 편집, 내보내기(프로)/삭제 |
| 5 | 결제 | 무료/프로 플랜 비교, 프로 구독 |

세션 이름 입력은 두지 않는다(불필요).

## 4. 디자인 시스템

- **톤:** 라이트 블루 글래스모피즘. 배경은 거의 단색에 가까운 연한 블루
  (`#f3f7fd`)로 절제하고, **유리감(반투명 blur)은 카드에만 은은하게** 적용.
- **포인트 컬러:** 단색 블루 `#3b82f6` (그라데이션 과용 금지).
- **타이포:** 굵은 숫자 중심(측정값 강조). system-ui.
- 다크 모드는 추후.

## 5. 오디오 코어 (DSP 파이프라인) — 네이티브 C++

- **입력:** AAudio, 48kHz / mono / float(PCM_FLOAT) / UNPROCESSED
  (AGC·노이즈제거·자동게인 OFF, OS 보정 배제)
- **프레임:** 0.125초 = 6000 샘플 단위 처리
- **연산 순서:** A-가중 IIR 필터(biquad 캐스케이드) → 프레임 RMS →
  `dB(A) = 20·log10(RMS) + calibrationOffset`
- **집계:** Leq(1분 등가소음도), Lmax(구간 최고 0.125초 레벨)
- **캘리브레이션:** 외장마이크 기준 오프셋 상수 1개. 시리얼별 동기화는 다음 단계.

## 6. 순환 버퍼 & 이벤트 저장 — 네이티브

- **30초 raw PCM 링버퍼** 상시 유지 (고정 크기, 메모리 누수 방지)
- dB(A) 임계 초과 → **전 5초 + 후 5초 = 10초**를 WAV로 로컬 저장
  (앱 전용 외부저장소). 전/후 초·임계값은 설정 파라미터.
- 저장 직후 Kotlin 콜백 → Room에 이벤트 메타데이터 기록

## 7. 센서 이동감지

- Kotlin `SensorManager`로 가속도+자이로 구독, 임계 이상 움직임 시 측정 세션/
  이벤트에 **"이동됨" 플래그** 표시. 측정은 중단하지 않음.

## 8. 위치(GPS) & 지도

- **세션당 1회**, 측정 전 화면에서 위치 확보 (`FusedLocationProviderClient`).
  위치 권한 요청 + 동의, 거부해도 측정은 동작.
- 좌표를 **역지오코딩**해 주소 문자열도 함께 저장.
- **Google Maps SDK**로 준비/저장/상세 화면에 지도 미리보기 + 핀 표시
  (전 세계 대응). 한국은 상세도 제한, 중국은 미지원 → 필요 시 지역별 지도
  제공자 교체 가능한 구조로 추상화(지금은 Google Maps 단일).

## 9. 데이터 저장 (전부 로컬)

| 저장소 | 내용 | 형식 |
|---|---|---|
| Room (SQLite) | 세션/이벤트 메타데이터 | DB 테이블 |
| 앱 전용 외부저장소 | 원음 | WAV 파일 |

### Room 엔티티 (초안)

```
MeasurementSession
  id, startTime, endTime, durationSec
  maxLmax, avgLeq, eventCount
  latitude, longitude, address     // 세션당 1회 위치
  movedFlag, memo

NoiseEvent
  id, sessionId (FK)
  timestamp, peakDb (Lmax), leq, durationSec
  wavPath
  moved                            // 센서 이동 플래그
  tags: List<String>               // 사용자 + AI 추천(더미)
  note: String?
```

## 10. 다국어 (i18n)

- 앱 구조를 처음부터 `strings.xml` 리소스 분리로 다국어 대응.
- **초기 출시:** 한국어·영어 실제 번역. 나머지 언어는 출시 후 점진 추가.
- 진입 시 언어 선택 화면 + 설정에서 변경. 기기 언어 자동 감지를 기본값으로.

## 11. 수익화 (무료 / 프로)

출시 시점 기준, **로컬에서 실제 동작하는 것만** 과금 경계로 둔다.

| | 무료 (₩0) | 프로 (₩4,900/월) |
|---|---|---|
| 실시간 dB(A) 측정 | ✅ | ✅ |
| 이벤트 무제한 저장 | ✅ | ✅ |
| 태그·노트 | ✅ | ✅ |
| 원음 재생 | **하루 5회** | **무제한** |
| 원음 내보내기 | ❌ | ✅ |
| 광고 | 표시 | **제거** |

- **결제:** Google Play Billing (구독). **광고:** AdMob (무료 플랜에만).
- 핵심 전략: "측정·기록은 공짜로 신뢰 확보, 증거가 되는 원음의 무제한 재생·
  내보내기와 광고 제거를 프로로 과금."
- 클라우드·AI·레포트는 만들어지는 다음 단계에 프로에 추가하거나 상위 티어로.

## 12. 더미 AI 연동 (인터페이스만)

- 이벤트 상세에 **AI 추천 태그** 자리를 둔다.
- 녹음 완료 후 호출하는 인터페이스를 정의하고, 지금은 **더미(고정/규칙 기반)
  응답**으로 동작시켜 화면·데이터 흐름을 검증. 실제 모델은 추후.

## 13. 모듈 경계

| 모듈 | 책임 | 의존 |
|---|---|---|
| `audio-core` (C++) | AAudio 입력, A-가중 DSP, dB(A)/Leq/Lmax, 링버퍼, WAV 저장 | AAudio, NDK |
| `jni-bridge` | 네이티브 ↔ Kotlin 함수/콜백 | audio-core |
| `measurement` (Kotlin) | Foreground Service, 폴링 루프, 센서 구독, 이벤트 수신 | jni-bridge, SensorManager |
| `location` (Kotlin) | GPS 확보, 역지오코딩, 지도 표시 | FusedLocation, Maps SDK |
| `data` (Kotlin) | Room DB, 세션/이벤트 CRUD, WAV 파일 경로 관리 | Room |
| `billing` (Kotlin) | 무료/프로 상태, 구독, 재생 횟수 게이트, 광고 노출 | Play Billing, AdMob |
| `ai` (Kotlin) | 더미 AI 태그 추천 인터페이스 | data |
| `ui` (Compose) | 전 화면(언어·준비·측정·저장·목록·상세·결제) | 위 모듈들 |

## 14. 비기능 요구사항

- 장시간 상시 측정 시 메모리 누수·크래시 방지: 고정 크기 링버퍼, 네이티브
  버퍼 재사용, **Foreground Service**.
- 기기별 마이크 편차 최소화: 외장마이크 + 캘리브레이션 오프셋.
