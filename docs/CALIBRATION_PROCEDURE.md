# 외장마이크 캘리브레이션 절차

앱이 출력하는 dB(A)를 실제 음압레벨(SPL)에 맞추기 위한 보정 오프셋 산출 방법.
현재 `MeasurementService.CALIBRATION_OFFSET`(임시 `94.0f`)을 이 절차로 구한 실측값으로 교체해야
측정 신뢰도가 확보된다.

## 배경
- NDK 엔진은 마이크 입력의 RMS로 **dBFS**(full-scale 기준, 항상 ≤ 0)를 구한 뒤
  `dB(A) = 20·log10(rms) + CALIBRATION_OFFSET` 로 SPL로 환산한다.
- 따라서 `CALIBRATION_OFFSET` = (기준 SPL) − (그 음원에서 측정된 dBFS) 이다.
- 플레이스홀더 `94.0f`는 음향 캘리브레이터의 표준 출력 **94 dB SPL @ 1 kHz** 에서 유래한 추정치일 뿐,
  실제 외장마이크+기기 조합의 응답을 반영하지 않는다.

## 방법 A — 음향 캘리브레이터 사용 (권장, 가장 정확)
준비물: 1/2" 또는 1" 마이크용 **음향 캘리브레이터**(예: 94 dB SPL, 1 kHz 출력), 외장마이크 키트.
1. 캘리브레이터에 외장마이크를 정확히 삽입/결합한다.
2. 캘리브레이터를 켜고 94 dB SPL, 1 kHz 톤을 출력한다.
3. 앱(또는 디버그 빌드)에서 **현재 측정 dBFS**(오프셋 0 기준의 raw 값)를 읽는다.
   - 산출용으로 `CALIBRATION_OFFSET = 0.0f` 로 임시 빌드해 측정 화면의 값(=dBFS)을 읽거나,
   - 엔진의 `FrameProcessor`에 일시적으로 dBFS 로깅을 추가해 평균값을 확인한다.
4. `CALIBRATION_OFFSET = 94.0 − (측정된 dBFS)` 로 계산한다.
   - 예) dBFS가 −34.0이면 offset = 94 − (−34) = **128.0**.
5. 구한 값을 `CALIBRATION_OFFSET`에 적용하고, 캘리브레이터에서 다시 측정해 **94 ± 0.5 dB(A)** 로
   나오는지 검증한다.

## 방법 B — 기준 소음계 대조 (캘리브레이터 없을 때, 근사)
준비물: 신뢰할 수 있는 **소음계(Class 1/2)** 또는 검교정된 기준기.
1. 외장마이크와 기준 소음계를 **동일 위치·동일 거리**에 나란히 둔다.
2. 안정적인 정상 소음(예: 핑크노이즈 스피커)을 재생한다.
3. 기준 소음계의 dB(A) 값 `S`와, 앱의 (오프셋 0) dBFS 값 `F`를 동시에 여러 번 읽어 평균낸다.
4. `CALIBRATION_OFFSET = S − F`.
5. 서로 다른 음압(예: 50/65/80 dB)에서 반복해 오프셋이 일정한지(선형성) 확인한다. 편차가 크면
   마이크 비선형/클리핑 가능성 → 마이크/게인 점검.

## 시리얼별 캘리브레이션 (향후)
- 외장마이크 개체차가 있으면 마이크 **시리얼 넘버별 오프셋**을 관리해야 한다(스펙 2-3).
- 권장 구조: Firebase에 `calibrations/{serial}` 문서로 오프셋 저장 → 앱이 연결된 마이크 시리얼로
  동기화해 `CALIBRATION_OFFSET`을 런타임에 주입. (현재는 단일 상수, 후속 작업.)

## 검증 기준
- 캘리브레이터/기준기 대비 **±1 dB 이내** 권장.
- A-가중 필터 자체는 단위테스트로 IEC 61672 응답이 검증돼 있으므로(1kHz≈0dB, 100Hz≈−19dB),
  오프셋만 맞추면 절대 레벨이 정렬된다.
